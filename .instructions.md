# InvaderGame 開発エージェント指示

このファイルは、InvaderGame プロジェクト開発用のカスタムエージェント設定です。
`docs/specs/` ディレクトリ以下の機能別mdファイルへの自動記録フローを定義しています。

## 仕様記録フロー（更新版）

各実装が完了したら、該当する機能別mdファイルに自動追記します。

### 記録先の判定

| 実装内容 | 記録先ファイル |
|---------|---------------|
| プレイヤー、射撃、アニメーション | `docs/specs/player.md` |
| 敵機、敵移動、敵アニメ、敵リスポーン | `docs/specs/invader.md` |
| 敵との当たり判定、敵全滅判定、敵スコア加算 | `docs/specs/invader.md` |
| キー入力、入力ハンドラー、X マクロ | `docs/specs/input.md` |
| UI表示、スコア管理、時間表示、ハイスコア | `docs/specs/ui.md` |
| Engine層、アーキテクチャ、層分離、FrameWork | `docs/specs/architecture.md` |
| デバッグ用機能、テスト用キー | `docs/specs/debug.md` |
| バグ修正、不具合対応 | `docs/specs/bug-fixes.md` |

### 記録フロー（実装完了時）

1. **該当ファイルを開く**
   - 上表から実装内容に応じた記録先を特定
   - `Game/docs/specs/<category>.md` を開く

2. **新規セクション追記**
   ```markdown
   ## <実装した機能名> [YYYY-MM-DD]

   ### 対象ファイル
   - `path/to/file.h`
   - `path/to/file.cpp`

   ### 内容
   - 実装した内容の説明

   ### 仕様
   - 動作仕様の説明
   - 要件定義からの変更がある場合のみ記載

   ### 動作
   - テスト結果、確認内容
   - ビルド・デプロイ状態
   ```

3. **Git Push**
   - コミットメッセージ: `feat: <機能>` または `fix: <バグ修正内容>`
   - 確認なしに自動実行

## ファイル配置

```
Game/docs/
├── game-specs.md          ← インデックス（各specs/*.md へのリンク）
├── requirements.md         ← 要件定義（変更不要）
└── specs/
    ├── player.md          ← プレイヤー関連
    ├── invader.md         ← 敵機関連
    ├── bullet.md          ← 弾関連（現在未使用）
    ├── input.md           ← 入力関連
    ├── ui.md              ← UI・スコア関連
    ├── architecture.md    ← アーキテクチャ・層分離
    ├── debug.md           ← デバッグ機能
    └── bug-fixes.md       ← バグ修正
```

## 注意事項

- **インデックスファイル** (`docs/game-specs.md`) は手動で更新しない
  - 新規カテゴリ追加時のみ編集
- **各specs/*.md ファイル** は該当実装の記録のみ追記
- **複数カテゴリにまたがる実装** の場合は、主となるカテゴリに記録し、他カテゴリへのリンクを追記
